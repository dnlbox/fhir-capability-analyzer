# Spec 10 — CI

**Status:** open

## Goal

Set up GitHub Actions CI to run type checking, linting, and tests on every push and pull request.
Ensures the project stays green and contributors get fast feedback.

## Dependencies

- Spec 08 (examples-fixtures) complete — tests need fixtures to run

## Deliverables

| File | Description |
|------|-------------|
| `.github/workflows/ci.yml` | GitHub Actions CI workflow |

## Key interfaces / signatures

No code interfaces. This spec produces CI configuration.

## Implementation notes

### Workflow triggers

- Run on `push` to `main` branch.
- Run on `pull_request` targeting `main` branch.

### Jobs

**ci** (single job, keeps it simple for v1):

1. Checkout code.
2. Set up Node.js (match `.nvmrc` version).
3. Install pnpm.
4. Install dependencies (`pnpm install --frozen-lockfile`).
5. Run type checking (`pnpm typecheck`).
6. Run linting (`pnpm lint`).
7. Run tests (`pnpm test`).
8. Run build (`pnpm build`).

### Matrix

- Node.js version: match `.nvmrc` (single version for v1, can add matrix later).
- OS: `ubuntu-latest` only for v1.

### Caching

- Cache pnpm store for faster installs.

## Acceptance criteria

- Workflow file is valid YAML.
- Pushing to `main` triggers the workflow.
- All four steps (typecheck, lint, test, build) run and pass.
- A failing test causes the workflow to fail.

## Do not do

- Do not add deployment or publish steps — that is spec 11.
- Do not add coverage reporting to an external service.
- Do not add a matrix of multiple Node versions or OSes in v1.
- Do not add Slack or email notifications.
